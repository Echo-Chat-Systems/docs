# `get`

Retrieves a [**public user**](/protocol/definitions/users/data#public-user-structure) by their user ID (Ed25519 public key). This action does not require authentication and returns only non-sensitive information.

## Overview

The `get` action allows clients to fetch basic profile information about any user by providing their public signing key. It is intended for public display purposes (e.g., when viewing another user's profile).

}

---

<Danger>Errors</Danger>

| Message        | Description                                 | Possible Causes                                                       |
|----------------|---------------------------------------------|-----------------------------------------------------------------------|
| `not-found`    | The requested user could not be found.      | The user ID does not exist or is not known to the server.             |
| `malformed-id` | The provided user ID is not a valid format. | The ID is the wrong length, not base64url, or not a valid public key. |

---

<Note>Notes</Note>

- This action returns only public fields such as display name, avatar, status, and reputation.
- If the user does not exist, an [`error`](../error) message will be returned with `message: "User not found"`.
