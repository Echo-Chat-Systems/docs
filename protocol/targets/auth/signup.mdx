# `signup`

Registers a new user on the server. This is a one-time operation per identity. The user provides their keypair and basic public profile information. The server validates the input, stores the identity, and begins tracking reputation and other user-associated data.

---

## Client ‚Üí Server

```json
{
    "target": "auth",
    "data": {
        "action": "signup",
        "params": {
            "keys": {  ...  },
            "profile": {  ...  }
        }
    }
}
```

### Fields

- `keys`: A public keypair. See [`pubkeys`](/protocol/definitions/crypto/keys#pubkeys) for field details
- `profile`: A basic profile object. See [`/protocol/definitions/users/data#profile`](/protocol/definitions/users/data) for field details.

---

## Server ‚Üí Client (Success)

```json
{
    "target": "auth",
    "data": {
        "action": "signup",
        "data":{
            "user": {  ...  }
        }
    }
}
```
