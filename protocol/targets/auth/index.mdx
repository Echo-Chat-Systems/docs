# `auth`

The `auth` target handles signing users in and out.
All `signin-*` actions are part of a secure multi-step authentication process using Ed25519 keypairs, without requiring usernames or passwords.

---

## Available Actions

| Name                                                    | Description                          |
|---------------------------------------------------------|--------------------------------------|
| [`signin-start`](/protocol/targets/auth/signin.mdx)     | Initiates a signin flow.             |
| [`signin-challenge`](/protocol/targets/auth/signin.mdx) | Server challenge to the user.        |
| [`signin-response`](/protocol/targets/auth/signin.mdx)  | User's signed response to challenge. |
| [`signin-complete`](/protocol/targets/auth/signin.mdx)  | Finalizes signin with user info.     |
| [`signout`](/protocol/targets/auth/signout.mdx)         | Terminates the current session.      |
| [`signup`](/protocol/targets/auth/signup.mdx)           | Signs a user up to the server        |
